---

- **Dedicated CPU**
    - CPU가 **완전히 전용**으로 할당되는 서버.
    - CPU 성능이 일정하고 안정적이라 **데이터베이스, 웹 서비스, 트랜스코딩 등** CPU 집중 작업에 적합.
    - 다른 서버와 CPU를 공유하지 않으므로 예측 가능한 성능이 필요할 때 좋음.
        
- **Cloud GPU**
    - GPU가 장착된 서버.
    - **AI 학습, 딥러닝, GPU 가속 게임 서버, 영상 렌더링** 등에 사용.
    - 일반적인 웹 서버나 DB 용도로는 필요 없음.
        
- **Shared CPU**
    - CPU를 **다른 사용자가 공유**하는 가상 서버.
    - 가격이 저렴하고, 작은 웹사이트나 테스트 서버, 가벼운 애플리케이션용.        
    - 트래픽이 많거나 CPU 부하가 높은 작업에는 성능 변동이 있음.

- **Bare Metal**
    - 물리 서버 전체를 단독으로 사용하는 서버.
    - CPU, 메모리, 저장공간 모두 **완전히 독점** 사용 가능.
    - 높은 성능과 커스터마이징이 필요할 때 적합, 가격이 비쌈.

보여주신 이미지와 계획하신 **'Claude CLI + Python Socket + Web GUI + Docker'** 환경을 고려할 때, 가장 중요한 키워드는 **메모리(RAM)**와 **저장장치 속도(NVMe)**입니다.

결론부터 말씀드리면, **`vhp-1c-2gb-amd` ($12.00/mo)** 플랜을 강력하게 추천합니다.

---

## 1. 왜 이 플랜인가요? (선택 이유)

- **도커(Docker) 구동:** 도커 엔진 자체와 컨테이너(CLI, Socket, Web)를 여러 개 띄우려면 **최소 2GB RAM**이 필요합니다. 1GB는 운영체제와 도커만으로도 숨이 가빠져서 성능 저하가 발생할 수 있습니다.
    
- **Claude CLI & Python:** AI CLI 도구나 파이썬 서버는 의존성 라이브러리를 로드할 때 메모리를 꽤 사용합니다. 2GB가 되어야 스왑(Swap) 없이 쾌적하게 돌아갑니다.
    
- **AMD + NVMe (High Performance):** 두 번째 이미지의 AMD 라인업은 NVMe 스토리지를 사용합니다. 첫 번째 이미지의 일반 SSD($10짜리 vc2 모델)보다 **데이터 읽기/쓰기 속도가 압도적으로 빨라** 도커 빌드나 패키지 설치 시 체감이 큽니다.
    
- **Bandwidth (3 TB/mo):** 나중에 GUI를 통해 데이터를 주고받거나 외부 API와 통신할 때도 넉넉한 전송량을 제공합니다.

## 3. 핵심 체크포인트

1. **확장성:** Vultr은 나중에 클릭 몇 번으로 `1c-2gb`에서 `2c-4gb` 등으로 **스냅샷 없이 바로 업그레이드(Resize)**가 가능합니다. 따라서 처음부터 너무 무리하실 필요 없습니다.
    
2. **OS 선택:** 도커를 쓰신다면 **Ubuntu 22.04 LTS**나 **24.04 LTS**를 추천합니다. 커뮤니티 가이드가 가장 많습니다.
    
3. **위치(Location):** 서울(Seoul) 리전을 선택하셔야 CLI 명령을 내리거나 Web GUI를 조작할 때 지연 시간(Latency) 없이 빠릿하게 반응합니다.
