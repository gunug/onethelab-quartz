# WhisperX 자막 타이밍 해결 과정

  

## 문제

stable-ts 단독 사용 시 인스트루먼트 진입 구간에서 가사 타이밍 실패 (약 23%)

  

## 시도 1: WhisperX 순수 transcribe → align

- 결과: 10개 세그먼트 (여러 줄이 하나로 뭉침)

- 원인: VAD가 연속 보컬을 하나의 구간으로 감지

- 평가: 세그먼트 단위 타이밍은 정확하지만 줄 단위 자막 불가

  

## 시도 2: WhisperX에 가사 텍스트 직접 주입 (align만 수행)

- 결과: 전부 0.0초 (타이밍 없음)

- 원인: align은 세그먼트에 대략적 start/end가 있어야 범위 내에서 정밀 정렬 수행

- start=0, end=0으로 주면 정렬할 범위가 없음

  

## 시도 3: transcribe로 대략적 타이밍 획득 → 텍스트를 가사로 교체 → align

- 결과: 26줄 전부 타이밍 획득 (실패 0%)

- 문제: 반복 가사가 1번째 출현 위치로 끌려감

- 원인: phoneme 매칭이 동일 텍스트를 구분하지 못함

  

## 시도 4: Hybrid (stable-ts + WhisperX 실패 구간 보완)

- 결과: 실패율 0%

- 문제: 후처리 없이 합치면 시간 역전/겹침 발생, duration 비정상 (17초짜리 등)

- 근본 원인: 숫자상 모순은 해결하지만 실제 보컬 타이밍과 일치하는지 보장 못함

  

## 핵심 오해

**WhisperX의 세그먼트가 뭉쳐있는 것을 "인식 실패"로 오해했음**

  

실제로는:

- 세그먼트 내부에 **word-level 타이밍이 전부 있었음** (164개 단어, 모두 start/end 보유)

- 세그먼트가 뭉친 것은 VAD의 구간 감지 단위일 뿐

- word-level을 가사 줄에 올바르게 배분하면 26줄 전부 정확한 타이밍 획득 가능

  

## 해결: 세그먼트 내 word-level 배분

```

WhisperX 세그먼트 (덩어리 타이밍 + 내부 word 타이밍)

     ↓

세그먼트 내 단어 수 ≈ 가사 줄의 단어 수로 순차 매칭

     ↓

각 줄에 해당하는 word들의 min(start) ~ max(end) 할당

```

  

### 이 방식이 잘 되는 이유

- 세그먼트 타이밍은 VAD 기반이라 **보컬이 실제로 있는 구간**을 정확히 잡음

- word-level은 wav2vec2 phoneme 정렬이라 **단어 경계가 정밀**

- 세그먼트가 시간순이므로 **반복 가사도 자연스럽게 구분**됨

- 오인식 단어("diatoms" 등)가 있어도 **순서와 단어 수 기반 배분이라 영향 없음**

  

### 남은 문제

- WhisperX가 오인식한 텍스트는 무시하고 우리 가사를 그대로 사용 (해결됨)

- 세그먼트 경계에서 1~2초 오차 가능 (허용 범위)

  

## 결론

stable-ts, hybrid 방식보다 **WhisperX word-level 배분 방식이 가장 깔끔하고 정확**
