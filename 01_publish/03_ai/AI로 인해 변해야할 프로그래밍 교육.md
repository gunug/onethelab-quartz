
* 기존 개발자는 코드 중심의 전문성을 가지고 있었고,
* 신규 바이브코더는 서비스 중심의 목표를 가지고 들어온다.

* 이 시리즈는 프로그래머를 만드는 과정이 아니다.
* 이 시리즈는 소프트웨어 제작자를 만드는 과정이다.
* 우리는 문법부터 시작하지 않는다.
* 우리는 서비스부터 시작한다.

# 개발자의 역할
* 구: 문장 작성→ 함수 작성→ 클래스 작성→ 프로그램 작성
* 신: 문제 정의→ 구조 설계→ AI에게 전달→ 검증→ 수정

* 구조를 이해 > 요구사항 작성 > AI에게 구현지시 > 검증
* 코드 작성 능력(X) -> 명세 작성 능력, 구조화 능력(O)
* 코드를 작성하는 기술(X) -> AI에게 의도를 전달하는 기술(O)

# 구 개발자 교육
* 변수, 반복문, 함수
* 언어, 프레임워크, 라이브러리, IDE, 디자인 패턴
* 문법
	* 자료구조, 알고리즘, 네트워크, 데이터베이스, 웹개발, 서비스 제작
* 로그인 화면을 만든다DB를 연결한다...

# 신 개발자 교육
* 서버, 클라이언트, 네트워크, API, 인증
* AI 에이전트, 자동 생성, 멀티모달, MCP, 워크플로우
* 서비스 아이디어
	* AI와 프로토타입 제작, 사용자 반응, 운영, 필요한 지식을 역으로 학습
* 왜 회원가입이 필요한가?어떤 인증방식이 필요한가?구글 로그인만으로 충분한가?서버가 필요한가?
* 왜 내 앱은 로그인 기능을 넣으려는데 서버가 필요한가?
* 서버가 필요한가?
- 서버리스로 가능한가?
- OAuth를 쓸 것인가?
- 회원가입이 정말 필요한가?

## 신 개발자 교육 목표
- API가 무엇인지 안다.
- 서버가 왜 존재하는지 안다.
- 데이터가 어디에 저장되는지 안다.
- 인증이 왜 필요한지 안다.
- 배포가 무엇인지 안다.
- 비용이 어디서 발생하는지 안다.

"내가 왜 이 구조를 만들었지?"

# 변화한 개발자의 의미
* 개발자 = 코드 작성자 (X)
* 개발자 = 문제 해결자 (O)
* 개발자 = 소프트웨어 시스템 설계자 (O)
